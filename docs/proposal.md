# sshell 需求文档

**版本**：v0.1  
**日期**：2026-06-07  
**状态**：草稿

---

## 1. 背景与目标

### 1.1 背景

需要一个轻量、可跨平台运行的 SSH 客户端命令行工具，满足以下使用场景：

1. **独立使用**：开发者、运维人员在终端直接运行，快速连接多台服务器。  
2. **被 Qt 应用集成**：Qt 应用以独立子进程方式拉起该工具，终端窗口由工具自身管理，Qt 侧不做窗口嵌入。

### 1.2 目标

- 使用 Rust 开发，单文件可执行，跨平台分发无需安装。  
- 提供完整 PTY 终端仿真（含 ANSI、全屏程序支持）。  
- 在同一终端窗口内通过 TUI Tab 管理多个 SSH 会话。  
- 提供标准化错误码，便于外部程序（如 Qt）解析退出状态。

## 2. 范围

### 2.1 MVP 包含

| 功能 | 说明 |
|------|------|
| 建立 SSH 连接 | 通过命令行参数发起连接 |
| 断开连接 | 关闭当前 Tab 或退出全部 Tab 时断开 |
| 多 Tab 管理 | TUI 内部 Tab 栏，支持新建、关闭、切换 |
| 交互式终端 | 完整 PTY 仿真（ANSI 颜色、光标控制、vim/top 等全屏程序） |
| 标准化错误码 | 进程退出码区分不同错误类型 |

### 2.2 MVP 不包含（后续阶段）

- 会话保存 / 快速重连  
- 文件传输（SFTP / SCP）  
- 密钥认证（仅密码认证）  
- 一次性命令执行模式（`-c "cmd"`）  
- 会话日志 / 命令审计  
- 图形 UI（后续可选阶段）  
- IPv6 支持  
- Qt 窗口内嵌终端（Qt 仅以外部进程方式拉起）

## 4. 功能需求

### 4.1 启动与连接

#### 4.1.1 命令行格式

```
sshell <连接串> [<连接串> ...]
```

**连接串格式**：

```
用户名:密码@主机[:端口]
```

- `主机`：IPv4 地址或域名；**MVP 不支持 IPv6**。  
- `端口`：可选，默认 `22`。  
- 密码可明文出现在命令行（MVP 可接受）。  
- 每个连接串启动时对应一个独立 SSH 会话 Tab。

**示例**：

```bash
# 单连接，默认端口
sshell root:123456@192.168.1.1

# 指定端口
sshell admin:p@ssw0rd@10.0.0.1:2222

# 同时打开多个 Tab
sshell user1:pass1@hostA user2:pass2@hostB:2222
```

#### 4.1.2 主机指纹策略

- MVP 默认信任，**不校验**服务端指纹。

#### 4.1.3 认证方式

- MVP 仅支持**用户名 + 密码**认证。

### 4.2 终端

- 每个 SSH 会话分配独立 PTY（Unix：POSIX PTY；Windows：ConPTY）。  
- 终端类型上报为 `xterm-256color`。  
- 支持完整 ANSI/VT 转义序列（颜色、光标移动、清屏、滚动区等）。  
- 支持全屏程序（vim、top、htop、less 等）。  
- 终端尺寸变化（窗口 resize）时同步更新 PTY 尺寸。

### 4.3 多 Tab 管理

#### Tab 操作

| 操作 | 说明 |
|------|------|
| 新建 Tab | 在运行时通过快捷键输入新的连接串并建立连接 |
| 关闭当前 Tab | 关闭当前会话，连接断开 |
| 切换至下一 Tab | 快捷键切换 |
| 切换至上一 Tab | 快捷键切换 |
| Tab 标题 | 显示 `user@host:port` |

- 同一主机允许开启**多个独立 Tab**（多会话并存）。  
- 最后一个 Tab 关闭后程序退出。

#### 快捷键（草案，实现时可调整）

| 快捷键 | 操作 |
|--------|------|
| `Ctrl+T` | 新建 Tab（提示输入连接串） |
| `Ctrl+W` | 关闭当前 Tab |
| `Ctrl+→` / `Ctrl+Tab` | 切换到下一个 Tab |
| `Ctrl+←` / `Ctrl+Shift+Tab` | 切换到上一个 Tab |

> 快捷键需避免与常见终端/系统快捷键冲突，最终以实现为准。

### 4.4 错误处理与退出码

程序通过**进程退出码**向调用方报告结果：

| 退出码 | 含义 |
|--------|------|
| `0` | 正常退出（所有 Tab 由用户主动关闭） |
| `1` | 通用错误（未分类） |
| `2` | 命令行参数错误（格式非法、缺少字段等） |
| `3` | 网络错误（主机不可达、连接超时） |
| `4` | 认证失败（用户名/密码错误） |
| `5` | SSH 协议错误 |
| `6` | 终端初始化失败（PTY 创建失败等） |

- 错误信息同时输出到 **stderr**，格式：`[ERROR] <错误码>: <可读描述>`。  
- 多 Tab 场景下，单个 Tab 断开仅在该 Tab 内提示，不影响其他 Tab；全部 Tab 均异常退出时程序返回首个错误码。

## 5. 非功能需求

### 5.1 平台支持

| 平台 | 最低版本 |
|------|----------|
| Windows | Windows 10（依赖 ConPTY，需 1809+） |
| Linux | Ubuntu 20.04（内核 5.4+） |
| macOS | macOS 15 (Sequoia) |

### 5.2 性能

- 支持**同时 20 个 SSH 会话**并发活跃。  
- 终端输出延迟（服务端 → 屏幕刷新）≤ 50ms。

### 5.3 分发

- 每个平台产出**单文件可执行文件**，无需运行时依赖（静态链接或内嵌依赖）。  
- 文件命名：`sshell`（Unix）/ `sshell.exe`（Windows）。

### 5.4 可维护性

- 采用 Rust 异步运行时（Tokio）管理多会话并发。  
- SSH 协议层与 TUI 层分离，便于后续添加图形 UI 或 Qt 集成接口。

## 6. Qt 集成说明

Qt 应用通过以下方式使用 sshell：

1. 调用 `QProcess::start("sshell", args)` 以独立进程启动，参数传入连接串。  
2. sshell **自行管理终端窗口**，Qt 不嵌入 UI 控件。  
3. Qt 通过监听 **进程退出码** 判断连接状态（参见 §4.4）。  
4. 如需主动关闭连接，Qt 通过 `QProcess::terminate()` / `kill()` 发送信号。

> 后续阶段如需 Qt 内嵌终端 UI，将另立需求文档评估 IPC 协议方案。
